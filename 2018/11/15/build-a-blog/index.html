面向小白的超详细hexo+github博客搭建教程

发表于 2018-11-15 | 分类于 hexo

前言

接触Hexo也是机缘巧合，10月份的时候在朋友圈刷到eleven小朋友撰写的个人博客搭建教程，文中提到了几个关键词：免费、高效、简洁、自由，这极大的激发了我的探索欲，便马上行动起来。当然程序员的教程对小白来说实在很不友好，过程中不可避免地出现了许多问题，不得不用各种姿势网上寻找解决方法。好在经历了多次推倒重建，反复尝试之后，终于成功了。喜悦和成就感无处安放，所以借此机会写一篇面向小白的实用教程，只需要会操作电脑，再多一点耐心和细心，拥有真真正正属于自己的博客就是这么简单~

搭建流程概述

  • 开通GitHub并创建项目仓库 —> 获得免费的远程仓库以托管本地仓库的代码
  • 安装git和node.js —> 安装hexo前的准备
  • 安装hexo —> 建立本地仓库，搭建博客框架
  • 配置SSH key —> 关联本地仓库和远程仓库
  • 部署到GitHub —> 将网站部署到服务器上
    注：电脑是win7 64位

准备Github

了解GitHub

推荐阅读GitHub官网操作指南中文翻译版

注册GitHub账号

1、进入GitHub官网，点击右上角的sign up（注册）跳转到注册页面或直接填写官网主页右侧的表单；

注意：用户名请花点心思取，后续将通过 https://username.github.io 访问你的博客；
2、点击按钮确定；

3、选择账号类型，我们直接选免费版，所有的代码将公开，然后点击继续；

4、根据自己的情况填写问卷提交或者直接跳过这一步；

5、到此，账号基本注册完成，接下来验证邮箱。登录邮箱，点击按钮确认；

6、现在可以开始使用GitHub了！

新建仓库

1、在右上角点击“＋”，在弹出的下拉选项中选择“New repository”；

2、将仓库信息补充完整，Repository name格式为：username.github.io（username是你注册时取的用户名）；

3、到此，仓库已经建好，将在此存放你的博客代码。

配置Github Pages

1、点击setting进入设置页面；

2、把页面往下滑找到GitHub Pages，按下图所示设置；

3、现在我们已经基于github创建好了可访问的博客空间，接下来需要填充内容。如果你有编程能力，可以编写网页文件，将其命名为index.html，上传到GitHub个人网站项目上，与其关联的样式文件或其他媒体文件上传也是同样的方法；

准备git

了解git

向自己的GitHub仓库上传文件需要借助“git”，那什么是git呢？推荐大家阅读廖雪峰老师的史上最浅显易懂的Git教程作一个初步的了解。

下载

进入git官网下载安装包；

安装

1、双击安装包开始安装，保持默认配置，每一步都点击“Next”即可，安装位置可自由选择，这里以D:\Program Files\Git为例；

2、安装完成后，单击鼠标右键，出现git右键菜单表示安装成功。

设置git用户名和email

1、设置git用户名和email，用来标记身份、匹配GitHub账户。打开右键菜单中的Git Bash，在弹出的命令窗口输入下列指令，按下“Enter”键确定；


2、输入下列指令按回车可查看当前设置的用户名和邮箱。

准备node.js和npm

安装node.js

1、Hexo是一款基于Node.js的静态博客框架，因此需先安装node.js，进入node.js官网下载安装包；

2、双击安装包开始运行，前面几步直接点击“Next”，安装路径可自行修改，这里以D:\Program Files\Git为例，单击“Next”，出现安装模式界面；

3、选择add to path后点击“Next”，这样安装时会自动添加环境变量；

4、继续点击next—>Install—>finish完成安装；

5、检测PATH环境变量是否配置了node.js，点击开始（电脑左下角图标），输入cmd，点击回车运行cmd.exe;

6、在弹出的窗口中输入命令path，点击回车后输出如下结果，可以看到环境变量中已经包含了D:\Program Files\nodejs\；

查看npm

node.js已经自带了npm，打开git bash输入npm -v可以查看安装的npm版本;

安装cnpm

1、npm安装插件是从国外服务器下载，受网络影响大，一个解决办法是使用npm的淘宝镜像（cnmp），官方网址传送门；
2、输入以下命令安装cnpm；


3、输入cnpm -v检测是否可以正常使用，输出下图所示内容表示配置成功；

4、如果输入cnpm -v出现的是“cnpm不是外部命令”，表示系统变量path并没有包含cnpm的路径，找到cnpm.cmd所在文件夹位置，将其路径添加到系统变量中的path里；

5、到此，安装hexo前的准备工作已经全部完成，因为使用npm的淘宝镜像代替了原生npm，所以后面所有涉及到npm的命令全都使用cnpm！

安装和配置hexo

认识hexo

hexo官方文档是这样描述的：“Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。”简而言之，hexo提供了博客框架以及将文档转换成静态网页的简便方法，使得普通人也可以快速建立网站，并且它还提供了相当丰富的主题样式，根据自己的喜好自由配置即可。

安装hexo

打开git bash，输入cnpm install -g hexo-cli安装hexo；

初始化hexo

1、新建一个文件夹用来存放hexo的代码，这里以D:\hexo为例，然后运行下列命令进行初始化；

3、输入cnpm install安装组件；

4、输入hexo g生成静态文件；
5、输入hexo s启动服务；

6、现在可以打开http://localhost:4000/进行本地预览了，出现下图证明hexo部署成功；

配置SSH keys密钥

目的

github要求推送代码的用户是合法的，所以每次都要输入账号密码进行验证，如果将git生成的公共SSH密钥添加到github中进行授权，则可直接提交代码无需再输入用户名和密码，配置SSH可以简单理解为二者签订了一份绿色通行协议；

检查SSH keys

输入ls -al ~/.ssh检查是否已经有公共SSH密钥，下列输出表示还没有；

配置SSH

1、使用git创建新SSH密钥，输入ssh-keygen，连续按三次回车即可生成SSH密钥；

2、用记事本打开id_rsa.pub，复制里面的内容；

3、登录github，点击右上角头像，在弹出的下拉框中点击settings进入设置；

4、点击SSH and GPG keys，再点击New SSH key按钮；

5、将复制过来的id_rsa.pub里面的内容粘贴到下面的输入框中；

6、重新登录github后，发现SSH keys下多了一条内容；

7、打开git bash，输入ssh -T git@github.com，检验SSH是否配置成功；

部署到GitHub

hexo提供了非常便捷的一键部署功能，现在我们已经可以通过git将搭建好的网站部署到服务器（github）上，但在此之前，还需要对hexo做一些基本的配置；

安装插件

输入cnpm install hexo-deployer-git --save，安装部署工具hexo-deployer-git；

一键部署
